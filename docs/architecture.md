# Ben HQ Architecture Notes

## Recommended Stack

- Next.js with TypeScript
- Tailwind CSS
- Supabase Auth
- Supabase PostgreSQL
- Vercel hosting
- Vitest for business logic
- Playwright for smoke tests

## Feature Folders

Organize product logic by feature:

```text
features/dashboard
features/tasks
features/calendar
features/training
features/pokemon
features/prompts
features/learning
features/notes
```

Shared UI should live in `components/ui`, layout in `components/layout`, and adapters in `lib/integrations`.

## Adapter Pattern

External data should flow through source-specific adapters into normalized internal records:

```text
External source -> provider adapter -> normalized model -> database/cache -> UI module
```

Every imported item should track:

- Source name
- Source URL or identifier
- Fetched timestamp
- Effective timestamp
- Freshness status
- Confidence or completeness when relevant

## Training Safety

Training plans should be generated by deterministic rules first. AI may explain, summarize, or suggest substitutions, but should not override hard constraints automatically.

Hard defaults:

- Saturday quality run
- Sunday long run
- Two weekly lifts
- No hard lower-body lift immediately before quality run
- No abrupt mileage increases
- Pain flag creates an easier/rest recommendation

## Integration Phasing

Start with manual and seeded data. Add read-only integrations only when the local workflow is useful.
